Potential Errors, Issues, and Flaws in blocking_mpmc_unbounded implementation:

1. Compilation & Include Issues:
   - MISSING INCLUDE: 'std::cout' and 'std::endl' are used in 'impl.hpp' but <iostream> is not included.
   - FIX: Add #include <iostream> or remove debug prints (recommended).

2. Logical & Concurrency Issues:
   - REDUNDANT CHECK: In 'wait_and_get()', the 'if(empty())' check after 'cond.wait()' is redundant because the predicate ensures the queue is not empty upon return.
   - WAIT_AND_POP(T&) SILENT FAILURE: If 'wait_and_get()' returns 'nullptr', 'wait_and_pop' returns without notifying the caller or modifying the reference.
     - FIX: Ensure 'wait_and_get' only returns 'nullptr' on shutdown, and have 'wait_and_pop' handle it (e.g., throw or return bool).
   - ATOMICITY VS LOCKING: 'size_q' is atomic, but its increment/decrement should stay within the locks to ensure consistency for observers who might hold one of the locks. (Currently correct, but must be maintained).

3. Performance & Design Flaws:
   - STACK OVERFLOW RISK: Recursive destructor calls via 'std::unique_ptr<node> next' will crash on large queues.
     - FIX: Implement a manual destructor: 'while(head) { head = std::move(head->next); }'.
   - FALSE SHARING: 'head_mutex' and 'tail_mutex' are adjacent. Multiple threads will fight for the same cache line.
     - FIX: Use 'alignas(std::hardware_destructive_interference_size)' for both mutexes.
   - UNNECESSARY COPIES: 'push(T value)' causes multiple copies/moves.
     - FIX: Provide 'void push(const T&)' and 'void push(T&&)', and implement 'emplace(Args&&... args)'.
   - DEBUG OVERHEAD: 'std::cout' inside 'cond.wait' significantly slows down the queue.
   - SHARED_PTR OVERHEAD: Using 'shared_ptr' for every node's data adds an extra heap allocation.
     - FIX: Consider storing 'T' directly or using 'std::unique_ptr' if ownership is transferred.

4. API & Feature Gaps:
   - MISSING 'emplace': Requested by user but not implemented.
   - MISSING 'static_assert': Should verify 'T' is not a reference and is movable.
   - CONST CORRECTNESS: 'empty()' and 'size()' are not 'const'.
   - SIZE TYPE: 'int' is used for 'size_q'; 'std::size_t' is preferred for container sizes.
   - COPY/MOVE CONTROL: The queue should explicitly delete copy/move constructors/assignments.

5. Minor Issues:
   - NAMESPACE POLLUTION: 'using namespace tsfqueue::__impl;' and 'using queue = ...' in headers will cause conflicts in user code.
     - FIX: Remove 'using namespace' from headers and move 'queue' alias inside the proper namespace.
   - UNUSED HELPER: 'get_tail()' is private and never used.
